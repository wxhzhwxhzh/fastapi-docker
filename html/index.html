什么是Docker？

Docker是一个开源的容器化平台，让你能够将应用程序和依赖项打包到标准化单元（容器）中，以确保应用在任何环境中都能一致地运行。

容器化技术

容器是轻量级的虚拟化单元，包含应用程序及其所有依赖项，确保应用在任何环境中都能以相同的方式运行。

标准化部署

Docker容器提供了一致的运行环境，解决了"在我的机器上能运行"的问题，简化了开发到生产的流程。

资源高效

与传统虚拟机相比，容器共享主机操作系统内核，启动更快，占用资源更少，可以在同一台主机上运行更多容器。

Docker容器 vs 传统虚拟机

Docker容器

  • 共享主机OS内核
  • 启动时间：秒级
  • 资源占用：低
  • 镜像大小：MB级别
  • 隔离级别：进程级隔离

传统虚拟机

  • 需要完整OS
  • 启动时间：分钟级
  • 资源占用：高
  • 镜像大小：GB级别
  • 隔离级别：完全隔离

安装Docker

选择适合你操作系统的安装指南，按照步骤完成Docker的安装配置。

Windows 安装步骤

1

检查系统要求

Windows 10 64位：专业版、企业版或教育版（Build 16299或更高版本）

确保已启用Hyper-V和容器功能

2

下载Docker Desktop

从Docker官网下载Windows版本的Docker Desktop

3

安装Docker

运行安装程序，按照提示完成安装

安装过程中会要求启用WSL 2，按照指引完成配置

4

验证安装

打开命令提示符或PowerShell，运行以下命令验证安装：


    

docker --version
docker run hello-world

macOS 安装步骤

1

检查系统要求

macOS 10.15 Catalina或更高版本，支持Intel或Apple芯片

2

下载Docker Desktop

从Docker官网下载macOS版本的Docker Desktop

根据你的芯片选择相应版本（Apple芯片或Intel芯片）

3

安装Docker

将Docker拖放到Applications文件夹中

从Applications文件夹启动Docker，首次启动会要求输入密码

4

验证安装

打开终端，运行以下命令验证安装：


    

docker --version
docker run hello-world

Linux 安装步骤（以Ubuntu为例）

1

卸载旧版本

如果系统中已安装旧版本Docker，先卸载：


    

sudo apt-get remove docker docker-engine docker.io containerd runc
2

安装必要的依赖


    

sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release
3

添加Docker官方GPG密钥


    

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
4

设置Docker仓库


    

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
5

安装Docker引擎


    

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
6

验证安装


    

sudo docker --version
sudo docker run hello-world

可选：配置非root用户使用Docker，避免每次输入sudo：


    

sudo usermod -aG docker $USER

配置完成后需要注销并重新登录才能生效

Docker基础命令

掌握这些核心命令，轻松管理Docker镜像和容器。

docker pull

从Docker仓库拉取镜像


    

docker pull [镜像名:标签]
# 示例
docker pull ubuntu:22.04

docker images

列出本地所有镜像


    

docker images
# 或
docker image ls

docker rmi

删除本地镜像


    

docker rmi [镜像ID或名称]
# 强制删除
docker rmi -f [镜像ID或名称]

docker build

根据Dockerfile构建镜像


    

docker build -t [镜像名:标签] [Dockerfile路径]
# 示例
docker build -t myapp:1.0 .

docker run

创建并启动容器


    

docker run [选项] [镜像名] [命令]
# 示例：交互式运行ubuntu
docker run -it ubuntu:22.04 bash

docker ps

列出正在运行的容器


    

# 列出运行中的容器
docker ps
# 列出所有容器（包括停止的）
docker ps -a

docker start/stop

启动/停止容器


    

# 启动容器
docker start [容器ID或名称]
# 停止容器
docker stop [容器ID或名称]
# 强制停止容器
docker kill [容器ID或名称]

docker exec

在运行的容器中执行命令


    

docker exec [选项] [容器ID或名称] [命令]
# 示例：进入容器的bash
docker exec -it mycontainer bash

docker rm

删除容器


    

docker rm [容器ID或名称]
# 强制删除运行中的容器
docker rm -f [容器ID或名称]
# 删除所有停止的容器
docker container prune

docker network ls

列出所有网络


    

docker network ls

docker network create

创建自定义网络


    

docker network create [网络名称]
# 示例：创建桥接网络
docker network create my-network

docker volume ls

列出所有数据卷


    

docker volume ls

docker volume create

创建数据卷


    

docker volume create [卷名称]
# 示例
docker volume create my-data

docker volume rm

删除数据卷


    

docker volume rm [卷名称]
# 删除未使用的数据卷
docker volume prune

Docker实战示例

通过实际案例学习如何使用Docker部署应用程序。

使用Docker部署Nginx服务

步骤1：拉取Nginx镜像


    

docker pull nginx:latest

步骤2：创建并启动Nginx容器


    

docker run -d \
  --name my-nginx \
  -p 8080:80 \
  nginx:latest

参数说明：
-d：后台运行容器
--name my-nginx：指定容器名称为my-nginx
-p 8080:80：将主机的8080端口映射到容器的80端口

步骤3：验证Nginx服务

打开浏览器，访问 http://localhost:8080，你应该能看到Nginx的默认欢迎页面。

或者使用curl命令验证：


    

curl http://localhost:8080

步骤4：自定义Nginx页面

创建一个自定义HTML页面，并通过数据卷挂载到容器中：


    

# 创建本地目录和HTML文件
mkdir -p ./nginx/html
echo "<h1>Hello from Docker Nginx!</h1>" > ./nginx/html/index.html

# 启动容器并挂载目录
docker run -d \
  --name my-custom-nginx \
  -p 8080:80 \
  -v $(pwd)/nginx/html:/usr/share/nginx/html \
  nginx:latest

现在访问 http://localhost:8080 就能看到你的自定义页面了。

步骤5：停止和删除容器


    

# 停止容器
docker stop my-nginx

# 删除容器
docker rm my-nginx

使用Docker部署Python应用

步骤1：创建Python应用

创建一个简单的Flask应用：


    

# 创建项目目录
mkdir docker-python-app
cd docker-python-app

# 创建app.py文件
cat > app.py << EOF
from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello():
    return "Hello from Docker Python!"

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
EOF

# 创建requirements.txt
echo "flask==2.0.1" > requirements.txt

步骤2：创建Dockerfile


    

cat > Dockerfile << EOF
# 使用官方Python镜像作为基础
FROM python:3.9-slim

# 设置工作目录
WORKDIR /app

# 复制依赖文件并安装
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY app.py .

# 暴露端口
EXPOSE 5000

# 启动命令
CMD ["python", "app.py"]
EOF

步骤3：构建镜像


    

docker build -t my-python-app:1.0 .

步骤4：运行容器


    

docker run -d \
  --name python-app-container \
  -p 5000:5000 \
  my-python-app:1.0

步骤5：验证应用

访问 http://localhost:5000 或使用curl命令：


    

curl http://localhost:5000

使用Docker部署MySQL数据库

步骤1：拉取MySQL镜像


    

docker pull mysql:8.0

步骤2：创建数据卷（可选）

创建数据卷用于持久化MySQL数据：


    

docker volume create mysql-data

步骤3：启动MySQL容器


    

docker run -d \
  --name mysql-container \
  -p 3306:3306 \
  -v mysql-data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=mysecretpassword \
  -e MYSQL_DATABASE=mydb \
  -e MYSQL_USER=myuser \
  -e MYSQL_PASSWORD=mypassword \
  mysql:8.0

参数说明：
-e MYSQL_ROOT_PASSWORD：设置root用户密码
-e MYSQL_DATABASE：创建初始数据库
-e MYSQL_USER和MYSQL_PASSWORD：创建普通用户及密码

步骤4：连接到MySQL


    

# 方式1：通过容器内的mysql客户端
docker exec -it mysql-container mysql -uroot -p

# 方式2：通过本地mysql客户端
mysql -h localhost -P 3306 -uroot -p

步骤5：查看数据库日志


    

docker logs mysql-container

常见问题

解答Docker初学者常遇到的问题。

Docker容器和传统虚拟机的主要区别在于资源利用率和启动速度。虚拟机需要运行完整的操作系统，而容器共享主机的操作系统内核，只包含应用程序及其依赖。这使得容器更轻量、启动更快（秒级），并且占用更少的资源。此外，容器的隔离级别低于虚拟机，但对于大多数应用场景已经足够。
可以使用docker logs命令查看容器日志： 
docker logs [容器ID或名称]
若要实时查看日志，可以添加-f参数： 
docker logs -f [容器ID或名称]
在Docker中，有几种方式可以在容器之间共享数据： 1. 使用数据卷（Volumes）：创建一个数据卷，然后挂载到多个容器中。 2. 使用绑定挂载（Bind Mounts）：将主机上的目录同时挂载到多个容器中。 3. 使用临时文件系统（tmpfs mounts）：在容器的内存中创建临时文件系统，可用于临时共享数据。 推荐使用数据卷的方式，因为它由Docker管理，更安全可靠。
Docker提供了便捷的命令来清理未使用的资源： 
# 清理所有未使用的资源（镜像、容器、网络、卷）
docker system prune

# 清理所有未使用的镜像（包括悬空镜像）
docker image prune -a

# 清理所有停止的容器
docker container prune

# 清理未使用的数据卷
docker volume prune
注意：这些命令会永久删除数据，请确保你不再需要这些资源。
Docker Compose是一个用于定义和运行多容器Docker应用程序的工具。使用YAML文件配置应用程序的服务，然后使用单个命令创建和启动所有服务。 使用步骤： 1. 创建docker-compose.yml文件定义服务。 2. 运行docker compose up启动所有服务。 3. 运行docker compose down停止并删除服务。 Docker Compose特别适合开发环境和小型部署，简化了多容器应用的管理。
